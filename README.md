# VERSION-CONTROL-WITH-GIT

## Objective
This project demonstrates Git branching and merge conflict resolution.


## Files Included
- main.py
- feature.txt
- conflict_demo.txt
- README.md


## Git Workflow Steps

### 1. Create Repository
Project is initialized on GitHub.

### 2. Create Branch
feature-branch is created from main.

### 3. Modify Files
Both branches modify the same file:
conflict_demo.txt

### 4. Merge Conflict
When merging feature-branch into main, a conflict occurs.

### 5. Conflict Resolution
Manually resolved by combining both changes.

Final content:
This file is merged from MAIN and FEATURE branch
Conflict successfully resolved


## Outcome
- Branching demonstrated
- Merge conflict created
- Conflict resolved successfully
- Clean Git workflow completed
